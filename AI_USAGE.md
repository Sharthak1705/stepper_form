# AI Usage Documentation

## Overview

Artificial Intelligence tools were used during the development of this project to accelerate implementation, improve debugging efficiency, and assist with architecture decisions.

The AI assistance was used as a development support tool only. All generated code, architecture decisions, validations, and integrations were manually reviewed, modified, tested, and validated before being included in the final application.

---

# Areas Where AI Was Used

## 1. Backend Development

AI assistance was used for:

* Express.js route generation
* PostgreSQL migration support
* CRUD endpoint implementation


---

## 2. Frontend Development

AI assistance helped with:

* React component structuring
* Form stepper implementation
* State management patterns
* Validation handling
* API integration logic
* UX improvements for multi-step forms
* Unsaved changes warning flow

---

## 3. Database Design

AI support was used for:

* PostgreSQL table schema generation
* SQL debugging
  
---

* React

---

# Human Review & Validation

All AI-generated suggestions were:

* manually reviewed
* tested locally
* debugged when necessary
* modified to fit project requirements
* validated against application behavior

No AI-generated code was used without developer verification.

---

# Development Responsibility

Final implementation decisions, debugging, testing, architecture, and deployment responsibility remained fully under the developer's control.

AI was used strictly as a productivity and assistance tool during development.

---

# Notes

This project demonstrates practical integration of AI-assisted software development while maintaining manual engineering oversight and validation throughout the implementation lifecycle.
